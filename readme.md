# 💸 Mini Expense Tracker – JavaScript Debugging Case

## 📌 Overview

This project started as a **buggy and poorly structured beginner-level expense tracker**.

The goal was not just to build an app, but to **debug, refactor, and improve an existing codebase**, simulating a real-world frontend scenario.

Instead of starting from scratch, the focus was on:

* identifying issues
* fixing bugs
* improving UX
* restructuring logic
* extending functionality

---

## ⚠️ Initial State (Problem)

The original project had several issues:

* All JavaScript logic was written in a single file (no structure)
* CSS used overly generic selectors (`div`, `input`, `button`)
* No clear folder structure
* Poor UX (no feedback, no validation)
* Multiple functional bugs

---

## 🐛 Bugs That Were Identified & Fixed

* ❌ Incorrect total calculation (string concatenation instead of numbers)
* ❌ Form could be submitted with empty inputs
* ❌ Amount field accepted invalid values (text, negative numbers)
* ❌ No error or success feedback
* ❌ Enter key did not trigger form submission
* ❌ No visual input states (focus, error, success)

---

## 🚀 Features Implemented

The application was significantly improved with the following features:

### Core Features

* ✅ Add expense (name, amount, category)
* ✅ Real-time input validation
* ✅ Total expense calculation
* ✅ Data persistence using `localStorage`

### Advanced Features

* ✅ Delete expense
* ✅ Edit expense (inline editing)
* ✅ Category system with color tags
* ✅ Search by name (live filtering)
* ✅ Filter by amount range
* ✅ Filter by category
* ✅ Sorting (amount & name, ascending/descending)

---

## 🎨 UI / UX Improvements

* Improved layout and spacing
